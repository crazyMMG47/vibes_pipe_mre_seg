# vibes_pipe

End-to-end deep learning pipeline for **3-D MRE (Magnetic Resonance Elastography) brain segmentation**.  
Trains a Probabilistic U-Net on human-annotated GT masks and NLI-derived pseudo-labels, with scanner-aware noise augmentation for GE and Siemens scanners.  
Includes a full-stack **Prediction Viewer GUI** for reviewing MC-sampled predictions and curating pseudo-GT.

## 2. Installation

### 2.1 Create the conda environment

The project uses a single conda environment named **`torch3090`**.

```bash
# If the environment does not exist yet, create it:
conda create -n torch3090 python=3.9 -y
conda activate torch3090
```

### 2.2 Install PyTorch

Match the CUDA version on your machine. For CUDA 11.8:

```bash
conda install -n torch3090 pytorch torchvision torchaudio pytorch-cuda=11.8 -c pytorch -c nvidia -y
```

For CUDA 12.1:
```bash
conda install -n torch3090 pytorch torchvision torchaudio pytorch-cuda=12.1 -c pytorch -c nvidia -y
```

### 2.3 Install project Python dependencies

```bash
conda activate torch3090
pip install -e .                        # installs vibes_pipe as an editable package
pip install -r gui/backend/requirements.txt   # GUI backend deps
```

Required packages (installed by the above):
- `monai`, `scipy`, `numpy`, `opencv-python`, `tqdm`
- `fastapi`, `uvicorn[standard]`, `pillow`, `python-multipart`

### 2.4 Install Node.js (GUI only)

```bash
conda install -n torch3090 nodejs=20.17.0 -y
```

### 2.5 Install frontend npm dependencies

```bash
conda run -n torch3090 npm install --prefix gui/frontend
```

### 2.6 Verify

```bash
conda activate torch3090
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
python -c "import monai; print('monai', monai.__version__)"
python -c "import fastapi; print('fastapi ok')"
node --version   # should show v20.x
```

## 4. Data preparation

### 4.1 Input data requirements

Each subject needs:

| File | Description |
|---|---|
| `<id>_t2stack.mat` | 3-D T2 MRI volume (float, any shape) |
| `<id>_mask.mat` | Binary segmentation mask (uint8 or float) |
| `<id>_t2stack.nii` | NIfTI companion — used to read voxel spacing for resampling |
| `<id>_Mu.mat` | NLI stiffness output *(optional)* |
| `<id>_noise_profile.mat` | Scanner noise field *(optional, for noise augmentation)* |

### 4.2 pairs.json — the input spec

Create one `pairs.json` per experiment. Each entry is a subject record:

```json
[
  {
    "id": "G009",
    "split": "train",
    "scanner_type": "GE",
    "t2stack":    "/abs/path/to/G009_t2stack.mat",
    "t2stack_nii":"/abs/path/to/G009_t2stack.nii",
    "GT(human)":  "/abs/path/to/G009_mask.mat",
    "NLI_output": "/abs/path/to/G009_Mu.mat",
    "eligible_preds": [],
    "noise_profile": null,
    "meta": {}
  }
]
```

`split` must be one of `"train"`, `"val"`, `"test"`.  
`scanner_type` must be `"GE"` or `"SIEMENS"` (case-insensitive).

### 4.3 Run workspace prep

The prep step copies all files into a reproducible workspace layout and writes `manifest.json`:

```bash
conda activate torch3090

python -m src.vibes_pipe.cli.pipeline_cli prep \
  --pairs_json experiments/<exp_name>/pairs.json \
  --out_dir    experiments/<exp_name>/workspace_root
```

Options:
- `--overwrite` — allow overwriting an existing workspace
- `--no_hash`   — skip SHA-256 hashing (faster, less integrity checking)

After this step the workspace will contain:

```
workspace_root/
  manifest.json
  train/<id>/t2stack.mat  GT(human).mat  NLI_output.mat …
  val/<id>/…
  test/<id>/…
```

`manifest.json` is the single source of truth for all downstream steps.

## 5. Training

### 5.1 Choose a config

| Config | Mode | Notes |
|---|---|---|
| `configs/probabilistic_train.yaml` | Probabilistic | Prior + posterior, KL warmup, `inject_latent: true` |
| `configs/deterministic_train.yaml` | Deterministic | Plain U-Net + Fcomb head, `inject_latent: false` |

Edit the config to point at your manifest and set output paths:

```yaml
data:
  manifest_path: experiments/<exp_name>/workspace_root/manifest.json

trainer:
  save_path:         checkpoints/my_run_best.pt
  final_save_path:   checkpoints/my_run_last.pt
  history_save_path: checkpoints/my_run_history.json
```

### 5.2 Run training

```bash
conda activate torch3090

python -c "
from src.vibes_pipe.training.builders import build_engine
from src.vibes_pipe.data.builders import build_data
from src.vibes_pipe.utils.config import load_config

cfg = load_config('configs/probabilistic_train.yaml')
data = build_data(cfg)
engine = build_engine(cfg)
engine.fit(data['train_loader'], data['val_loader'])
"
```

Or use the notebook `usage.ipynb` which has the same flow with cells you can run step-by-step.

### 5.3 What training produces

```
checkpoints/
  <run>_best.pt      ← best val-Dice checkpoint: {epoch, model_state_dict, optimizer_state_dict, val_dice}
  <run>_last.pt      ← final epoch checkpoint
  <run>_history.json ← {train/loss_total, train/loss_recon, train/loss_kl, val/dice, val/loss, train/beta, train/lr}
```

### 5.4 Probabilistic vs deterministic mode

| Setting | Effect |
|---|---|
| `model.kwargs.inject_latent: true` | Prior + posterior networks active; KL term in loss; MC sampling at inference |
| `model.kwargs.inject_latent: false` | Vanilla U-Net path; no KL; single deterministic prediction per forward pass |

### 5.5 Loss tuning

```yaml
loss:
  kwargs:
    beta_final: 0.2      # max weight on KL term
    beta_warmup: 80      # epochs to ramp beta from 0 → beta_final
    free_bits: 0.0       # set >0 to prevent posterior collapse
    recon_loss:
      kwargs:
        alpha_combo: 0.0   # 0.0 = pure Dice; 1.0 = pure Focal; 0.2 = 20% Focal + 80% Dice
```

## 6. Inference

### 6.1 Edit the inference config

```yaml
# configs/infer/infer_1.yaml
inference:
  device: cuda
  checkpoint_path: checkpoints/my_run_best.pt
  save_dir: output/my_run_results
  n_mc_samples: 6       # number of MC forward passes (probabilistic mode)
  calculate_metrics: true
  save_predictions: true
  threshold: 0.5
  subject_shapes_json_path: null   # optional: path to JSON with original image shapes for resizing back

# Reuse the same data/preprocess/dataloader sections from your training config
data:
  manifest_path: experiments/<exp_name>/workspace_root/manifest.json
```

### 6.2 Run inference

```bash
conda activate torch3090

python -c "
from src.vibes_pipe.infer.builders import build_inference_engine
from src.vibes_pipe.utils.config import load_config

cfg = load_config('configs/infer/infer_1.yaml')
engine, meta = build_inference_engine(cfg)
print(f'Loaded checkpoint epoch={meta[\"epoch\"]}, val_dice={meta[\"val_dice\"]:.4f}')

summary = engine.run_inference(
    n_mc_samples=cfg['inference']['n_mc_samples'],
    calculate_metrics=cfg['inference']['calculate_metrics'],
    save_predictions=cfg['inference']['save_predictions'],
    threshold=cfg['inference']['threshold'],
)
print(summary)
"
```

### 6.3 What inference produces

For each subject in the test split, a subfolder is written to `save_dir`:

```
output/my_run_results/
  <subject_id>/
    prob_map.npy       ← mean predicted probability map [C, H, W, D] float32
    prob_map.mat       ← same, in .mat format
    pred_mask.npy      ← thresholded binary mask [H, W, D] uint8
    pred_mask.mat      ← same, in .mat format
    mc_samples.npy     ← all K probability maps [K, C, H, W, D] float32
    meta.json          ← {id, scanner_type, dice, n_mc_samples, mean_entropy, ged, …}
  summary.json         ← aggregate metrics across all subjects
```

These files are read directly by the GUI.

## 7. Prediction Viewer GUI

The GUI lets you browse inference results per subject, compare all MC samples against GT, inspect metrics, and export the best sample as pseudo-GT.

### 7.1 Configure

```bash
cp gui/.env.example gui/.env
```

Edit `gui/.env`:

```env
OUTPUT_DIR=/abs/path/to/output/my_run_results   # InferenceEngine save_dir
MANIFEST_PATH=/abs/path/to/workspace_root/manifest.json
WORKSPACE_ROOT=/abs/path/to/workspace_root
PORT=8000
```

| Variable | Required | Effect if missing |
|---|---|---|
| `OUTPUT_DIR` | **Yes** | Server won't start |
| `MANIFEST_PATH` | No | Raw image, GT, and stiffness slices disabled |
| `WORKSPACE_ROOT` | No | Pseudo-GT export disabled |
| `PORT` | No | Defaults to 8000 |

### 7.2 Run — development (recommended)

```bash
./gui/run_dev.sh
```

- Backend at `http://localhost:8000` — auto-reloads on Python changes
- UI at **`http://localhost:5173`** — hot module replacement
- Stop with `Ctrl-C`
- API docs (Swagger) at `http://localhost:8000/docs`

### 7.3 Run — production (single port)

```bash
./gui/run_prod.sh
```

Builds React then serves everything from `http://localhost:8000`.

### 7.4 Using the viewer

1. **Select a subject** in the left sidebar — shows Dice (green ≥ 0.8 / amber / red), scanner badge, stiffness indicator
2. **Scrub slices** with the slider or `←` `→` arrow keys; switch orientation (Axial / Sagittal / Coronal)
3. **Toggle GT Contour** to overlay the ground-truth boundary on every panel
4. **Review the candidate grid**: Input | GT | Sample 1 … Sample K
5. **Inspect metrics**: stat cards (Mean Dice, Entropy, GED) + per-sample Dice bar chart
6. **Select a candidate**: click a sample column *or* click its bar in the chart (amber highlight appears)
7. **Export**: click **"Export as Pseudo-GT"** — writes the chosen mask to `<workspace_root>/<split>/<id>/pseudo_mask.mat`
8. **Stiffness**: when `NLI_output.mat` is present, a stiffness panel (hot colormap) appears to the right, synchronized to the same slice index

### 7.5 Pseudo-GT in next training run

After exporting, re-train with `label_mode: prefer_pseudo` so the dataset automatically uses pseudo masks where they exist and falls back to GT otherwise:

```yaml
data:
  manifest_path: experiments/<exp_name>/workspace_root/manifest.json
  label_mode: prefer_pseudo
```

## 8. Configuration reference

### Model kwargs (`model.kwargs`)

| Key | Type | Default | Description |
|---|---|---|---|
| `image_channels` | int | 1 | Input image channels |
| `mask_channels` | int | 1 | Mask channels (for posterior) |
| `latent_dim` | int | 6 | Latent space dimension |
| `feature_channels` | list | [16,32,64,128] | U-Net encoder channel progression |
| `num_res_units` | int | 2 | Residual units per block |
| `inject_latent` | bool | true | `false` = deterministic mode |

### Loss kwargs (`loss.kwargs`)

| Key | Default | Description |
|---|---|---|
| `beta_final` | 0.2 | Max KL weight |
| `beta_warmup` | 80 | Epochs to ramp beta |
| `free_bits` | 0.0 | Free-bits threshold (prevents collapse) |
| `recon_loss.kwargs.alpha_combo` | 0.0 | 0 = pure Dice, 1 = pure Focal |

### Trainer kwargs (`trainer`)

| Key | Default | Description |
|---|---|---|
| `num_epochs` | 100 | Total epochs |
| `grad_clip` | 1.0 | Gradient clipping norm |
| `fast_val` | true | Validate on 1 batch only (fast epoch cycle) |
| `log_every` | 1 | Print log every N epochs |

### Preprocessing (`preprocess`)

| Key | Default | Description |
|---|---|---|
| `target_spacing` | [1.5,1.5,1.5] | Isotropic voxel spacing after resampling (mm) |
| `target_size` | [128,128,64] | Final spatial dimensions [H,W,D] |
| `clahe.enabled` | true | Contrast-limited adaptive histogram equalization |
| `clahe.clip_limit` | 2.0 | CLAHE clip limit |
| `normalize.mask_positive_only` | true | Normalise using foreground voxels only |

## 9. NLI / stiffness integration

NLI (Non-Linear Inversion) derives tissue stiffness from MRE wave images. The pipeline treats NLI output as:

1. **Pseudo-label source** — `NLI_output.mat` can be used directly as a label via `label_mode: pseudo` or `prefer_pseudo`.
2. **Stiffness overlay** — displayed in the GUI alongside segmentation predictions once `NLI_output.mat` exists in the workspace.

**Current status:** The GUI gracefully shows a "Not yet available" placeholder when stiffness is absent. No code change is needed — once `NLI_output.mat` is placed in the workspace and the manifest `files.NLI_output.dst` field points to it, the stiffness panel activates automatically.

## 10. Troubleshooting

| Problem | Fix |
|---|---|
| `OUTPUT_DIR env var is required` | Create `gui/.env` from `.env.example` and set paths |
| `ModuleNotFoundError: vibes_pipe` | Run `pip install -e .` from repo root with `torch3090` active |
| Backend 500 on slice endpoint | Check `MANIFEST_PATH` and `WORKSPACE_ROOT` are set and point to existing files |
| GUI shows "No inference results" | `OUTPUT_DIR` must point to `InferenceEngine.save_dir` (contains subject subfolders with `meta.json`) |
| Raw / GT slices show "Not available" | Set `MANIFEST_PATH` and `WORKSPACE_ROOT` in `gui/.env` |
| Stiffness always "Not yet available" | Run NLI to produce `NLI_output.mat`; verify `files.NLI_output` is in `manifest.json` |
| CUDA out of memory during training | Reduce `batch_size`, `feature_channels`, or `target_size` in config |
| `fast_val: true` but val Dice is suspiciously stable | Expected — fast_val uses 1 batch. Set `fast_val: false` for reliable val metrics (slower) |
| `npm: command not found` | `conda install -n torch3090 nodejs=20.17.0 -y` |
| Port 8000 already in use | Change `PORT=` in `gui/.env` or `kill $(lsof -ti:8000)` |
| Workspace prep fails on missing file | Verify all `src` paths in `pairs.json` exist on disk |
| SHA-256 mismatch during prep | Delete the conflicting file in `workspace_root` and re-run prep, or use `--overwrite` |
